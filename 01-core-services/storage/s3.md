# Amazon S3 — Simple Storage Service

Object storage — infinitely scalable, highly durable (11 nines) storage for any type of data.

## Cloud Storage Overview

AWS offers three storage categories — pick based on how your compute accesses the data:

| Type | Description | AWS Services |
|------|-------------|-------------|
| **Block storage** | Raw storage volumes attached to instances; OS sees it as a disk | EBS, Instance Store |
| **File storage** | Shared file system accessed over a network protocol (NFS/SMB) | EFS, FSx |
| **Object storage** | Flat namespace, accessed via HTTP API; unlimited scale | S3, S3 Glacier |

## What is S3?

- Stores data as **objects** inside **buckets**
- An object = file + metadata; max object size = **5 TB**
- Bucket names are **globally unique** across all AWS accounts
- S3 is a **regional** service but bucket names exist in a global namespace
- Provides **strong read-after-write consistency** for all operations (since Dec 2020)
- Designed for **99.999999999% (11 nines) durability** — data replicated across ≥3 AZs (for most classes)

## Storage Classes

| Class | Use Case | Retrieval | Availability |
|-------|----------|-----------|-------------|
| **S3 Standard** | Frequently accessed data | Immediate | 99.99% |
| **S3 Intelligent-Tiering** | Unknown/changing access patterns; auto-moves between tiers | Immediate | 99.9% |
| **S3 Standard-IA** | Infrequently accessed, needs fast retrieval | Immediate | 99.9% |
| **S3 One Zone-IA** | Infrequent access, can be recreated; single AZ | Immediate | 99.5% |
| **S3 Glacier Instant Retrieval** | Archive accessed a few times/year | Milliseconds | 99.9% |
| **S3 Glacier Flexible Retrieval** | Archive, 1–2 times/year | Minutes–hours | 99.99% |
| **S3 Glacier Deep Archive** | Long-term retention (7–10 years) | ≤12 hours | 99.99% |

> **Cost order (cheapest to most expensive):** Deep Archive < Flexible < Instant < One Zone-IA < Standard-IA < Intelligent-Tiering < Standard

## Lifecycle Policies

Automate transitioning objects between storage classes or deleting them:
- Example: Standard → Standard-IA after 30 days → Glacier after 90 days → delete after 365 days
- Applied at bucket or prefix/tag level
- Cannot move objects *up* a tier (e.g. Glacier → Standard) via lifecycle — only downward transitions

## Bucket Policies & Access Control

- **Block Public Access** — account/bucket-level setting; overrides any policy granting public access. Enable by default.
- **Bucket Policy** — resource-based JSON policy; controls cross-account access and public access
- **IAM Policies** — identity-based; attached to users/roles
- **ACLs** — legacy; AWS recommends disabling ACLs and using policies instead
- **Pre-signed URLs** — grant time-limited access to a specific object without requiring AWS credentials

## Versioning, MFA Delete & Object Lock

- **Versioning** — keeps all versions of an object; protects against accidental overwrites/deletes
- **MFA Delete** — requires MFA to delete object versions or suspend versioning; only root account can enable
- **Object Lock** — WORM (Write Once Read Many); compliance and governance modes; set at bucket or object level
  - **Compliance mode** — no one, including root, can delete/modify until retention expires
  - **Governance mode** — users with special permissions can override

## Encryption

| Method | Key Management | Notes |
|--------|---------------|-------|
| **SSE-S3** | AWS manages keys | AES-256; default since Jan 2023 |
| **SSE-KMS** | AWS KMS; you control key policy | Audit trail in CloudTrail; KMS API costs |
| **SSE-C** | Customer provides key per request | AWS does not store the key |
| **Client-side** | You encrypt before upload | Full control; more complexity |

## Performance & Transfer

- **Multipart upload** — recommended for objects >100 MB, required >5 GB; parallel upload parts
- **S3 Transfer Acceleration** — uses CloudFront edge locations to accelerate uploads over long distances
- **Byte-range fetches** — parallel downloads of specific byte ranges; speeds up large object downloads
- **S3 Select** — query CSV/JSON/Parquet objects with SQL; retrieve subset of data without downloading full object
- **Requester Pays** — requester pays data transfer costs, not bucket owner; useful for public datasets

## Static Website Hosting

- Enable on a bucket to serve HTML/CSS/JS directly from S3
- Combine with **CloudFront** for HTTPS, custom domain, global caching, and DDoS protection
- Use **Route 53** to route a domain to the CloudFront distribution

## Replication

- **Cross-Region Replication (CRR)** — replicate to a bucket in a different region; DR, compliance, latency
- **Same-Region Replication (SRR)** — replicate within same region; log aggregation, test/prod sync
- Versioning must be enabled on source and destination buckets

## Exam Tips

- S3 is **object** storage — not a file system, not a block device; cannot mount S3 like a disk
- Default: all S3 buckets are **private**; Block Public Access is on by default
- Versioning once enabled cannot be fully disabled — only suspended
- SSE-KMS = CloudTrail audit trail for key usage; choose when compliance requires key access logging
- Pre-signed URLs expire — use for temporary, controlled access (e.g. download links)
- Transfer Acceleration = long-distance uploads; CloudFront = content delivery/downloads
- Intelligent-Tiering has a monitoring fee per object — not cost-effective for tiny objects or very short-lived data
