# Amazon ElastiCache

Managed in-memory caching — Redis and Memcached engines.

## Key Topics

- **Redis vs. Memcached** — Redis: persistence, replication, Lua scripting, sorted sets, pub/sub; Memcached: simpler, multi-threaded, no persistence
- **Redis Cluster Mode** — Cluster Mode Disabled (single shard, up to 5 read replicas); Cluster Mode Enabled (multiple shards for horizontal write scaling)
- **Replication & Failover** — Redis Multi-AZ with automatic failover; primary + read replicas; Global Datastore for cross-region
- **Caching Strategies** — lazy loading (cache-aside), write-through, write-behind; TTL and eviction policies (LRU, LFU, volatile-lru)
- **Use Cases** — session stores, leaderboards (sorted sets), rate limiting, database query caching, real-time analytics
