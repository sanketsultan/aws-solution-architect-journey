# Lab 08 — Well-Architected Review

Conduct a Well-Architected Framework Review (WAFR) on a sample workload and document findings.

## Objectives

- Use the AWS Well-Architected Tool in the console to define a workload
- Answer the review questions for all six pillars
- Identify High-Risk Issues (HRIs) and Medium-Risk Issues (MRIs)
- Create an improvement plan and prioritise the top findings
- Document decisions and trade-offs in `wafr-notes.md`

## Prerequisites

- Access to the AWS Well-Architected Tool (no cost to use)
- A real or hypothetical workload to review (e.g., the three-tier web app from Project 01)

## Steps

1. Open AWS Console → Well-Architected Tool → Create Workload
2. Complete the profile (industry, environment, active regions)
3. Start a Review — answer all lens questions across six pillars
4. Review the generated HRI/MRI report
5. Create an improvement plan with target dates
6. Export the report (PDF) and paste key findings into `wafr-notes.md`
